		1. Коротко опишите, какие задачи с помощью механизма ветвления решаются в GIT? Зачем выполняется их слияние?
	
	Ветки(branches) - эта та концепция, которая считаетсяя основной в Git в системе контроля версий.
Ветвление решает ряд задач. Суть ветки это изолированный поток разработки, еще одна версия разработки , на которую можно 
переключиться, внести правки, потом переключиться на основную и избежать каких-либо проблем.
	Если работа  с правками закончена и её можно влить в ветку master. Для этого нужно выполнить слияние ветки. Все, что 
нужно сделать — переключиться на ветку, в которую вы хотите включить изменения, и выполнить команду git mergeю.
		Укажите команды для:

	-переименования веток:
git branch -m нов.название ветки

	-создания и переключения на новую ветку:
git dranch feature - создать новую ветку,
git checkout feature - переключиться на новосозданную ветвь;

git checkout -b login - создать нов. ветвь и сразу на нее переключиться;

	-удаление ветки:
git checkout master - переключаемся на главную ветвь, 
git branch -v  - смотрим какие есть ветки,
git branch -d название ветки - удаляем ненужную ветку,
git branch -D название ветки - удаляем ненужную ветку принудительно после сообщения с подсказкой:

	-слияния веток:
git branch
git diff ..название ветки - смотрим на разницу между ветками
git diff master..название ветки - смотрим какие изменения будут привнесены,
git merge название ветки - делаем слияние,
git branch -d название ветки - удаляем ненужную ветвь.
